# Case 006: Outbound Network Connection Over HTTPS

## Case Summary
In this investigation, we examined a Sysmon network connection event generated by PowerShell during an outbound HTTPS connection.

Network connection events are important in security monitoring because they show which process initiated the communication, which destination was contacted, and which protocol and port were used.

The goal of this investigation was to review the network connection event, identify the process responsible, and determine whether the activity was suspicious or part of a controlled lab test.

-----

## Detection Source

### Log Source
- Microsoft-Windows-Sysmon/Operational

### Event Type
- Sysmon Event ID 3 - Network Connection

Sysmon Event ID 3 records network connections associated with a process, including source and destination IP addresses, ports, and protocol information.

-----

-----

## Analysis

Outbound network connections are commonly reviewed by security teams to identify suspicious communication, unauthorized external access, malware beaconing, or payload retrieval.

In this case, PowerShell initiated a TCP connection to an external IP address over port 443, which is commonly used for HTTPS traffic.

PowerShell is a legitimate administrative tool, but it is also frequently abused by attackers for downloading content, executing scripts, and interacting with remote infrastructure.

This specific activity was intentionally generated in the cybersecurity monitoring lab using a controlled command to produce network telemetry for investigation.

Indicators confirming the activity is benign:

- The connection was manually initiated during a controlled lab exercise
- The activity used a standard HTTPS destination port
- No malicious follow-on execution was observed
- The event was generated for defensive monitoring practice

-----

## Outcome

The network connection event represents controlled lab activity and demonstrates how Sysmon captures outbound process-level network connections, including destination IP address, port, and protocol.
